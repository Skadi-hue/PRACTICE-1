Команда проекта

🔹 2 марта 2026 – Завершена разработка портативного компилятора C++ в JSON с графическим интерфейсом на C#

Команда VIDEOTEXT полностью переработала устаревший движок, привязанный к Qt. Проведён рефакторинг на стандартный C++17, создана консольная утилита s_compiler.exe и легковесная обёртка на C# Windows Forms. Реализован асинхронный запуск, автоопределение кодировки (UTF-8 / Windows-1251), Drag‑and‑Drop, пакетная обработка, встроенный JSON‑вьюер и история операций.
✅ Итог: выпущена Portable-версия приложения, работающая на любом Windows-ПК без установки дополнительных библиотек.

🔹 23 марта 2026 – В веб-приложение Syllabica интегрирован асинхронный рендеринг видео

Решена архитектурная проблема совмещения FastAPI-прототипа (асинхронный) и основного Flask-приложения (синхронное). Разработан модуль video_queue.py на базе очереди queue.Queue (паттерн Producer‑Consumer). Фоновый воркер выполняет рендеринг MP4 без блокировки серверных потоков. Добавлены API-эндпоинты: создание задачи, проверка статуса, скачивание готового видео.
✅ Итог: сервер перестал «зависать» при генерации тяжёлых роликов, обеспечена многопользовательская работа.

🔹 27 апреля 2026 – Запущен комплексный интерфейс для филологов: парсинг DOCX → JSON → Python-сценарии + табличное представление данных

Разработан полный цикл создания видеотекста без единой строчки кода.

  • Модуль docx_to_json.py извлекает из Word-таблиц операции с форматированием (жирность, цвет фона).
  • scenario_generator.py превращает JSON в исполняемый Python-сценарий (Pygame/FFmpeg).
  • Веб-интерфейс оснащён редактором Ace Editor и кнопками экспорта во все форматы.
  • Добавлено альтернативное табличное представление JSON (маршрут /table_view) для наглядной работы с иерархическими данными.
  • Внедрены пользовательские «песочницы» (изолированные папки по user_id) с защитой от подмены путей.
    ✅ Итог: время подготовки одного видеотекста сократилось с 1–2 часов до 10–15 минут, филологи получили интуитивно понятный инструмент.
