## Вариативная часть задания
Тема вариативной части: «Разработка Telegram-бота "Рандомайзер"»

Бот предназначен для генерации случайных значений по запросу пользователя, делает это строго и без интерпретаций: принимает диапазон чисел, список вариантов или условия, выдаёт один случайный результат. Проект реализован как утилитарный инструмент для быстрых случайных выборов в повседневных задачах.

### Структура репозитория
```
proekt-site/                 – корневая папка проекта
├── .git/                    – служебная папка Git (история версий)
├── .idea/                   – служебная папка редактора IntelliJ IDEA (настройки)
├── archetypes/              – шаблоны для новых страниц Hugo
├── assets/                  – исходные медиафайлы (CSS, JS, изображения) перед обработкой
├── content/                 – все страницы и посты сайта (Markdown)
│   ├── _index.md            – домашняя страница (аннотация)
│   ├── about.md             – страница «О проекте»
│   ├── contributors.md      – страница «Участники»
│   ├── resources.md         – страница «Ресурсы»
│   └── journal.md           – страница «Журнал» 
├── public/                  – собранная статическая версия сайта (для публикации)
├── static/                  – статические файлы (CSS, изображения, favicon)
│   └── css/
│       └── custom.css       – уникальные стили сайта
├── themes/                  – установленные темы Hugo
│   └── ananke/              – тема Ananke (подключена как Git-подмодуль)
├── .gitmodules              – конфигурация Git-подмодулей (тема Ananke)
├── .hugo_build.lock         – технический файл Hugo (блокировка сборки)
├── hugo.toml                – главный конфигурационный файл Hugo
└── README.md                – описание репозитория (этот файл)
```

## Отчётная часть

### Описание создания статического веб-сайта
### 1. Изучение основ веб-технологий

- Освоены базовые принципы HTML5 и CSS3: структура документа, блочная модель, позиционирование, flexbox, адаптивная вёрстка через медиа-запросы.
- Приобретены навыки создания простых статических страниц «на чистом HTML/CSS».

### 2. Установка и настройка Hugo

- Для удобства установки на Windows использован менеджер пакетов **Scoop**
- Создан новый сайт командой `hugo new site proekt-site`.
- В качестве темы выбрана **Ananke** – популярная, стабильная и хорошо документированная
- Cоздано пользовательское меню (Главная, О проекте, Участники, Журнал, Ресурсы)

### 3. Развёртывание на хостинге
- Выбрана платформа **Netlify** благодаря простоте использования
- После автоматического развёртывания сайт стал доступен по постоянному адресу:  
    **https://skadi-and-myata.netlify.app**.
